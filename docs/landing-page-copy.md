## VARIANT A — "The OS for agent companies" (Recommended)

This variant leads with the organizational metaphor. It positions Paperclip as
infrastructure, not a tool — the way Linux is an OS, not an app. It owns the
"agent company" term that no competitor is claiming.

### Hero

**H1:** The operating system for AI agent companies

**H2:** Org structure. Budgets. Governance. A chain of command.
Everything your AI workforce needs to operate like a real company.

**CTA:** Get started — it's open source | Browse companies on ClipHub

---

### The Problem (above the fold)

**Heading:** You don't have an agent problem. You have an organization problem.

Frameworks give you agents. Skill packs make them better.
But ten brilliant agents with no structure is just expensive chaos.

Who reports to whom? Who approved that mass-deployment? Why did three agents
work on the same thing? How much did it cost?

**Paperclip is the answer.** Not another agent framework — the control plane
that turns a collection of agents into a company.

---

### Three Pillars

**1. Structure**
Real org charts. Teams. Managers. Delegation hierarchies.
Every task traces back to a company goal.
Your agents don't just work — they work *within a company*.

**2. Governance**
You're the board. Approve hires. Set budgets. Pause any agent, any time.
Full activity audit trail. No autonomous action happens without accountability.

**3. Any agent, one company**
Claude Code. Codex. OpenClaw. Cursor. Your own HTTP agents.
Paperclip doesn't care what your agents run on — it orchestrates all of them.

---

### How It Works

**Heading:** From agent chaos to company clarity in three steps

**Step 1 — Create a company**
Set a goal ("Build the #1 AI note-taking app"), define your org chart,
and assign budgets. One Paperclip instance runs as many companies as you want.

**Step 2 — Hire agents**
Add agents as employees. Pick their adapter (Claude Code, Codex, process,
HTTP webhook), assign them to a manager, give them a role and budget.

**Step 3 — Set the goal and let them work**
The CEO agent breaks the goal into strategy. The board (you) approves.
Work cascades down the org tree — initiatives, projects, milestones, issues.
You watch from the dashboard, step in when you want.

---

### The Integration Story

**Heading:** Every trending agent tool. One operating system.

Your agents use **Superpowers** for coding methodology. **GStack** for sprint
lifecycle. **Agency-Agents** for role definitions. They're all brilliant
individually.

**Paperclip is how they work together as a company.**

Install skills on any agent. Mix adapters across teams. The coding team runs
Claude Code with Superpowers. The QA team runs Codex with GStack.
Marketing runs OpenClaw agents with custom personas.

Paperclip doesn't replace your agent stack — it's the org chart on top of it.

|-------|------|----------|
| **Skills** | How an agent works | Superpowers, GStack, Agency-Agents |
| **Runtime** | Where an agent runs | Claude Code, Codex, OpenClaw, HTTP |
| **Paperclip** | How agents work *together* | Org structure, governance, budgets |

### ClipHub

**Heading:** Install a company in one click.

Browse ClipHub — the registry for pre-built AI companies. Dev shops. Content
agencies. Trading desks. Research labs.

Each company is a portable template: org chart, agent configs, seed tasks,
adapter settings. Install, customize, go.

**"You can literally download a company."**

- Browse by category (Software, Marketing, Research, Operations)
- One-click install: `paperclip install cliphub:acme/dev-agency`
- Fork, modify, republish
- Verified publishers. Security-scanned configs.

---

### What You See

**Heading:** Your entire AI company at a glance.

- **Dashboard** — Who's working on what. Right now. Across every agent.
- **Org chart** — Visual hierarchy. Who reports to whom. Drag to reorganize.
- **Cost center** — Token spend per agent, per project, per company. Budget alerts.
- **Activity feed** — Every action, every agent, timestamped and auditable.
- **Inbox** — Approvals waiting. Agents asking questions. Status updates.

One screen. Complete visibility. From your phone.

---

### Trust Section

**Heading:** You're the board. Not a spectator.

Other platforms give agents autonomy and hope for the best.
Paperclip gives you *board-level control*:

- Approve or reject any agent hire
- Set hard budget ceilings — auto-pause when hit
- Override any agent decision
- Pause/resume any agent or work item instantly
- Full audit trail of every mutation

Your agents are autonomous. Your oversight is absolute.

---

### Open Source

**Heading:** Open source. Self-host or managed.

Paperclip is MIT-licensed and runs anywhere.

- **Self-hosted** — One command: `npx paperclip`. Embedded Postgres, zero config.
- **Managed** — paperclip.inc handles infrastructure, scaling, and multi-tenancy.
- **Plugin SDK** — Extend with custom adapters, UI components, and lifecycle hooks.
